Single-page synthesis after Phase 3–5: what we know, the (reality-adjusted) end goal,
and the path to get there. Supersedes scattered next-steps.

## 1. What we know for certain (validated this session)

**The edge:**
- Edge = **ML SELECTION within a gated population**, and it is **thin**.
- The raw deterministic signal population LOSES: structural setups / state-gated trades net
  **PF 0.6–0.8, WR 0.35–0.40, −0.2R/trade**, every pair, every year, even with NY gate.
- ML picking the top ~3% flips that to **PF 1.3–1.6** — the ML selection IS the entire edge.

**Where the edge lives (and where it doesn't):**
- Validated good only on **FX majors + NY session + ECN spread**.
- **FX is session-driven**: remove the NY gate and FX PF drops 1.27→1.11.
- **Does not generalize**: cross-asset (agnostic gate) PF — FX 1.11, crypto 1.05, metals
  0.95, indices 0.88 (mean 1.00, 0/4 classes ≥1.3). No universal edge exists.

**Best validated FX config (the one genuinely good, robust thing):**
- long (5 pairs) + USDCHF-short + META re-rank + POOLED top10 + proba sizing, 50 trees,
  ECN 0.5pip → **net PF 1.30 (1.52 sized), WR 0.51, ~9 trades/day, all years positive
  (2026 1.40–1.66), Pine budget 88%.**

**Levers that work / are dead (don't re-try the dead ones):**
- WORK: short side (density), pooled selection (lookahead-free ≈ hindsight), meta-labeling
  (+WR+PF), proba sizing (+PF, +variance), tree reduction to 50 (fits Pine, edge holds).
- DEAD: cutoff loosening, trend-align gate, dynamic exits (BE/trail), seed ensemble,
  sample-uniqueness weighting, short-specific features, TP/SL sweep (R=1.5 optimal),
  deterministic setups as a standalone edge, the universal single-model.

**Deployment reality:** Pine budget 5000 ops/bar. FX long+short = 88%, +META@50t = 88%.
Per-symbol indicator (no cross-symbol ranking needed). request.security 14/40.

## 2. The end goal — reality-adjusted (Nico-approved 2026-06-01)

Original aspiration: one indicator, **good results on any asset & any timeframe**, user-set
RR/ADR, backtest. **The data killed the "good results everywhere via one edge" part.**

**Accepted reframe — separate TOOL from EDGE:**
> **PaceAlgo = a universal, adaptive, self-calibrating signal + risk-management TOOL**
> (any asset/TF by construction, user-configurable RR/ADR, reproducible non-repaint
> backtest) **+ an optional ML "Confidence" overlay that delivers a real edge only where
> validated (FX first), expanding class-by-class via the ANN-009 router.**

- The **tool** is the universal, honest, sellable utility (no auto-edge promise).
- The **edge** is the premium AI overlay, FX-only today.
- "Good PF/WR" is a property of the **overlay**, not the tool. Market accordingly.

Honest truth: "good auto-results on every asset" is NOT achievable. "A solid universal
tool + a strong FX AI module + honest class-by-class AI expansion" IS.

## 3. How we get there — phased plan

**Phase 0 — Lock the proven asset (the spine). [highest value, lowest risk]**
Kernel-lock the FX config (primary long/short + meta, 50t) → snapshot thresholds/sizing/
gate → Pine export (4 cascades + feature engine + adaptive gate + R exec) → bit-exact +
ops<5000 verify. Outcome: one real, good, shippable thing (FX AI module).

**Phase 1 — Build the adaptive tool frame around it.**
Deterministic universal core in Pine: vol/trend context + **adaptive activity-window
detection** (replaces hardcoded NY) + user-selectable signal modes + **user RR/ADR risk**
+ **backtest panel** (PF/WR/MDD per chart/TF, non-repaint). FX AI module plugs in as the
"Confidence" overlay; off (or generic) on unsupported markets. Honest UX: tool everywhere,
AI badge where validated.

**Phase 2 — Industrialize per asset class (ANN-009 router).**
For each class, test with CLASS-APPROPRIATE structure (the untested fair test):
- **Indices first** — they trade a strong US cash/RTH session (analogous to FX-NY); the
  agnostic test that hurt them most (0.88) removed exactly that. RTH-gated indices are the
  most promising next AI module.
- Then crypto (own time/weekend effects), metals.
Add an AI Confidence overlay only for classes that clear the per-class ML bar (≥~1.3 net,
all-years-positive, walk-forward). Classes that don't clear it stay tool-only.

**Phase 3 — V1.5 / V2 backend (per original evolution).**
Monthly retrain → auto-ship new Pine; later cloud inference + webhook + dashboard.

**Revised launch gates (replace old §1.4 universal-edge premise):**
- (a) Tool bar: non-repaint verified, backtest reproducible bit-exact, clean UX, works on
  any chart without errors.
- (b) AI-overlay bar (per class): net PF target, all-years-positive, walk-forward robust,
  Pine bit-exact. Launch FX overlay when it clears; add classes as they clear.

**Immediate next action:** Phase 0 — lock + export the FX module. It de-risks everything
(gives a concrete, good product core) and is the substrate the tool frame plugs into.

## 5. V1-UX-Featureplan (Web-Session-Nachträge 2026-06-02 — Erkenntnisse GELOCKT)

**Gelockte Erkenntnisse (TP/SL-Philosophie):**
- Box-LÄNGE = Funktion von TP/SL-Abstand relativ zur Volatilität, NICHT vom Hebel.
  ATR-Normalisierung skaliert Boxen automatisch mit dem TF — Trading-Art steckt
  implizit im gewählten Timeframe.
- **HEBEL ≠ TP/SL-Platzierung.** Stop gehört an die strukturelle Invalidierung
  (Markt/Vola), Hebel ist reine Positionsgrößen-Entscheidung. Indikator kennt den
  Hebel nicht und koppelt TP/SL NIE daran (Swift Algo auch nicht).
- Zwei orthogonale Achsen sauber trennen: **Profile = Signal-SELEKTIVITÄT** (Strategy-
  Gruppe) vs. **Trading Style = Trade-HORIZONT/Box-Größe** (Risk-Gruppe) — UI-Labels
  entsprechend.
- **Frequenz/Qualitäts-Tradeoff als Designprinzip:** mehr Signale = verrauschter.
  Frequenz NIE als fixe Zahl hardcoden (8/Tag ≠ Ziel), sondern als User-Achse
  exponieren; ehrliches Tooltip-Framing. Net-PF vs. Frequenz beim Validieren messen
  (Phase-3-Logik auf den Regel-Core übertragen).

**Priorisierte Feature-Liste (NACH Core-Validierung auf Trend-Charts):**
1. **HOCH — Sensitivity-/Frequenz-Regler:** EIN Knopf skaliert ADX-Schwelle +
   Pullback-Strenge + adaptiven Multiplier zusammen. Empfehlungs-Panel schlägt pro
   Chart-TF eine Sensitivity vor (Swift-Tabelle-analog, aber regime-basiert).
2. **MITTEL — Multi-TP (TP1/TP2) + Teilausstieg + Break-Even-Stop:** TP1 ~1R sichern,
   SL auf Einstand, Rest läuft auf TP2=RR. Exit-GEOMETRIE (kein Fit) — hebt die
   angezeigte WR strukturell über 50–60% und ist der Haupt-Hebel für Nicos
   WR-Anforderung. Konkurrenz (SMRT) hat es.
3. **MITTEL — Trading-Style-Preset Scalp/Intraday/Swing:** dünne Komfort-Schicht
   (Struktur-Lookback + ATR-Band + RR-Default bündeln). *Status: Kern bereits in
   iter2 umgesetzt (Nico-Order 2026-06-02); RR-Default-Kopplung + Risk-Gruppen-Label
   noch offen.*
4. NIEDRIG — Trend-Stärke-Visualisierung (Zonen-Deckkraft nach Trendstärke).
5. NIEDRIG — MTF-Dashboard 4→8 TFs (4 reicht für V1).

## 6. PF-Verbesserungs-Leiter (CEO-Richtungsvorgabe 2026-06-02 — GELOCKT)

PF ist der Maßstab, WR ist Optik. Die ehrliche Leiter, in dieser Reihenfolge:

| Stufe | Hebel | Charakter |
|---|---|---|
| **1 (jetzt)** | **Exit-Geometrie:** Multi-TP (TP1 fix @1R — nie enger, §15-Lock) + Break-Even + Runner-Trail | Universell, ehrlich, verbessert die Payoff-VERTEILUNG. Erzeugt KEINEN Edge und wird nicht so vermarktet |
| 2 | **Selektion verschärfen:** MTF-Konfluenz vom Display zum harten Filter, ADX straffen | Weniger, bessere Trades |
| 3 | **Klassen-Spezialisierung (Router ANN-009):** pro Klasse ein walk-forward-validierter Parametersatz/Modul (siehe module_registry) | Validiert pro KLASSE, NIE per-Asset gefittet |
| 4 | **ML-Confidence-Overlay (V1.5):** 9-Feature-Ranker als Wahrscheinlichkeits-Filter auf Regel-Signale, nur wo Edge nachgewiesen (FX-Majors/NY) | ML rankt Signale, generiert keine |

**Die Verbesserungs-Maschine (Disziplin, jede Stufe):** EINE prinzipielle Hypothese pro
Iteration → out-of-sample über ≥4 diverse Assets → nur behalten wenn PF BREIT steigt
(nicht auf einem Asset) → dokumentieren. Kein Per-Asset-Tuning. (Musterbeispiel: das
echte HTF-Gate.)

**Ehrlichkeits-Boden:** Bleibt ein Asset trotz aller Stufen bei PF ~1.0 → WAIT-Panel,
dort nicht traden, keinen Edge faken. Ziel = „≥50% WR / PF >1.2 überall wo wir FEUERN",
nicht „auf jedem Asset erzwingen".

**Exit-Modell-Integrität (CEO-locked):** TP1 ≥ 1R (Sub-1R-Token-TP = WR-Inflation,
verworfen) · Win-Definition = Netto-R > 0 (TP1+BE = echter Win) · Panel zeigt Outcomes
GESPLITTET (TP2/Trail/BE/Loss), PF/Expectancy als Headline, WR nachgeordnet.
